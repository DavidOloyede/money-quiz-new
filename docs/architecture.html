Money Quizarchitecture explorer · updated 2026-06-15 for the Node.js backend (accounts, sync, Plaid, tickets, admin)

Money Quiz is a React + TypeScript frontend (Vite) with an optional Node.js backend — a Fastify API over PostgreSQL (Drizzle). Signed out, everything lives in the browser's localStorage and no server runs at all. Sign in (Supabase Auth handles login) and the same data syncs through the Node API, which also owns bank connections, support tickets, the admin panel, and activity logging. The one-minute version

Views (Import, Dashboard, Year Sheet, Quiz, Settings, plus Account & Admin) read everything from store.tsx, the single React context that owns the transactions and all remembered edits. The store persists to localStorage through storage.ts and delegates every calculation to pure helpers in src/lib — most importantly analysis.ts (totals, trends, recurring detection) and quiz.ts (question generation). When you sign in, cloudSync.ts mirrors each localStorage slice to the Node API (api.ts → server/), which stores it in PostgreSQL and also handles Plaid, tickets, admin, and events. Login itself is Supabase Auth.

Numbers

~40 frontend components · 21 lib modules (8 with unit tests) · 3 bundled data files · a Node (Fastify) API in server/ with 6 Postgres tables. Frontend: npm run dev; both: npm run dev:all; test: npm test.

There are two data layers. Locally, persistent state is JSON in the browser's localStorage (every key prefixed moneyquiz., defined in src/lib/storage.ts) — the map below shows those entities and how they reference each other. When you sign in, each localStorage key is mirrored to a row in PostgreSQL via the Node API (the user_slices table), alongside the server-only tables listed under the map (bank tokens, tickets, events). Click a box to see its fields, types, and where it's stored. The central entity is Transaction; almost everything else is a "memory" map that re-applies user edits to freshly imported transactions, keyed by one of four identity functions from src/lib/merchant.ts and src/lib/categorize.ts:

  • merchantKey(description)cleaned merchant identity
  • overrideKey(description)normalized exact description
  • groupKey(description, aliases)alias-aware display-name identity
  • txSignature(date, desc, amount)one specific charge
